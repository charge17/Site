# AI SEO Site

تطبيق PHP عربي خفيف لاستضافات Namecheap Stellar Plus لإدارة موقع مقالات SEO يعتمد على نماذج OpenRouter المجانية فقط، بدون Composer أو قاعدة بيانات إلزامية.

## المزايا الأساسية

- لوحة إدارة موسعة للنيشات، المقالات، الذكاء الاصطناعي، SEO، الإعلانات، السكريبتات، الصور، الصفحات الثابتة، الإضافات، المهام المجدولة، والأداء.
- توليد مقال SEO عبر سلسلة نماذج OpenRouter مجانية مع Failover تلقائي عند فشل أي نموذج.
- استخدام نماذج مختلفة قدر الإمكان: نموذج للتخطيط، نموذج للمقدمة و6 عناوين، نموذج لـ8 عناوين، ونموذج لـ6 عناوين مع الخاتمة.
- كل مقال يتضمن Meta Title وMeta Description وFAQ Schema وHowTo وBreadcrumb وArticle schema data وروابط داخلية وخارجية وCTA وجداول ومقارنات وقوائم.
- وضع أسئلة FAQ في قائمة انتظار لإنشاء مقالات داعمة وبناء شبكة مقالات داخلية.
- Cron آمن لمعالجة قائمة الانتظار، مع تحديث حالة المهمة إلى processing/done/failed.
- Sitemap وRobots.txt ديناميكيان، وملف `.htaccess` لإعادة الكتابة وتحسين الرؤوس الأمنية على Apache/cPanel.

## روابط الإدارة

- عند استخدام `public/` كجذر الموقع افتح لوحة الإدارة من `/admin/` أو `/admin.php`.
- عند نسخ محتويات `public/` إلى `public_html` سيبقى `/admin.php` متاحًا كمدخل مباشر مناسب للاستضافات المشتركة.

## التثبيت على Namecheap Stellar Plus

1. ارفع الملفات إلى الاستضافة واجعل `public/` هو جذر الموقع إن أمكن.
2. إن لم تستطع تغيير Document Root، انسخ محتويات `public/` إلى `public_html` وعدّل مسارات `require` بما يناسب مكان رفع مجلدات `src`, `cron`, `data`.
3. عيّن متغيرات البيئة `OPENROUTER_API_KEY`, `SITE_BASE_URL`, `CRON_TOKEN` من لوحة التحكم أو ملف `.htaccess` آمن.
4. امنح مجلد `data/` صلاحية كتابة، وتأكد من منع الوصول المباشر له عبر `data/.htaccess` إذا كان داخل المسار العام.
5. أضف مهمة Cron:

```bash
php /home/USER/path/to/cron/run.php
```

أو استدعاء HTTP آمن:

```bash
curl "https://example.com/cron.php?token=CRON_TOKEN"
```

## سير عمل الذكاء الاصطناعي

1. النموذج الأول ينشئ عنوان SEO قويًا، 20 عنوانًا فرعيًا، FAQ، الكلمات المفتاحية والوصف.
2. النموذج التالي يكتب المقدمة و6 عناوين فرعية.
3. النموذج الثالث يكتب 8 عناوين وسطية مع الجداول والمقارنات.
4. النموذج الرابع يكتب 6 عناوين أخيرة مع الخاتمة وCTA.
5. يتم دمج الأجزاء، إضافة الروابط الداخلية والخارجية، وحفظ بيانات Schema.
6. يتم إدراج أسئلة FAQ في قائمة الانتظار كمقالات مستقبلية لبناء شبكة مقالات داخلية.
7. إذا فشل نموذج مجاني يتم الانتقال تلقائيًا للنموذج المجاني التالي من إعدادات `config.php` وتسجيل الحدث في `data/logs/ai-failover.log`.
